# Baby Tracker — понятный гид по экранам

Это приложение помогает родителям быстро записывать всё важное о малыше: кормления, сон, прогулки, подгузники, лекарства и другое.

Ниже — описание каждого экрана простыми словами, как для обычного пользователя.

## 1) Главный экран

Главный экран — это ваш «пульт управления» на каждый день.

Что здесь есть:
- **Название профиля малыша** вверху.
- **Индикатор режима хранения данных**:
  - «Синхронизировано» — если есть авторизация и данные отправляются в облако.
  - «Локальное хранилище» — если приложение работает только на этом устройстве.
- Кнопки перехода на другие экраны:
  - **Профиль малыша**
  - **Уведомления**
  - **Статистика**
- Блок **«Сегодня»** с краткой сводкой по текущему дню (количество событий и длительность/объёмы, где применимо).
- Блок **активных таймеров** (если, например, идёт сон, прогулка или кормление).
- Сетка кнопок активностей, чтобы быстро добавить новую запись:
  - Кормление грудью
  - Бутылочка
  - Сон
  - Купание
  - Прогулка
  - Активность
  - Отрыжка
  - Подгузник
  - Лекарство
- Раздел **«Последние записи»** (последние 10):
  - показывает время, детали, заметки;
  - можно **редактировать** или **удалять** запись.

---

## 2) Экран добавления/редактирования активности

Открывается после нажатия на тип активности с главного экрана или по кнопке редактирования.

Общее поведение:
- Можно указать время и дополнительные детали.
- Можно оставить **комментарий/заметку**.
- В Telegram-версии сохранение делается через главную кнопку внизу.

Что можно заполнить по типам:

### Кормление грудью
- Таймер отдельно для **левой** и **правой** груди.
- Можно вводить минуты вручную.
- Время начала.

### Бутылочка
- Время начала.
- Тип питания: грудное молоко / смесь / вода.
- Количество в мл.

### Сон, Прогулка, Активность
- Можно использовать **таймер** (старт/стоп) или ввести время вручную.
- Время начала и окончания.

### Купание
- Время начала и комментарий.

### Подгузник
- Время.
- Тип: мокрый или грязный.

### Лекарство
- Время.
- Название лекарства.

### Отрыжка
- Время.
- Цвет.
- Консистенция.
- Объём (меньше/больше чайной ложки).

## 3) Экран «Профиль малыша»

Здесь хранится информация о ребёнке и динамика роста.

Что можно делать:
- Заполнить и сохранить:
  - имя малыша;
  - дату рождения;
  - ссылку на фото (если используется).
- Добавлять измерения роста:
  - дата измерения;
  - вес;
  - рост.
- Редактировать или удалять ранее добавленные измерения.
- Смотреть историю измерений и простые графики по весу и росту.

---

## 4) Экран «Статистика»

Экран для анализа режима за неделю.

Что показывает:
- **Навигация по неделям** (можно смотреть прошлые недели).
- **Таблицу-таймлайн** по дням и часам, где видно, в какое время какая активность преобладала.
- **Цветовые индикаторы** по типам активностей.
- **Сводку за неделю** по каждому типу активности:
  - сколько раз в среднем в день;
  - средняя длительность в день (где есть длительность).

---

## 5) Экран «Уведомления»

Нужен, чтобы не забывать о важных делах. Уведомления приходят через Telegram-бота.

Что можно делать:
- Смотреть список уже созданных уведомлений.
- Включать/выключать уведомление одним переключателем.
- Создавать новое уведомление:
  - для выбранного типа активности;
  - по **времени** (например, каждый день в 09:00);
  - или по **интервалу** (например, каждые 3 часа);
  - выбирать дни недели;
  - задать свой заголовок и текст.
- Редактировать и удалять уведомления.

---

## Как пользоваться приложением каждый день (очень коротко)

1. Откройте **главный экран**.
2. Нажмите нужную активность (например, «Сон» или «Бутылочка»).
3. Заполните данные и сохраните.
4. Периодически проверяйте:
   - **Статистику** — чтобы видеть режим;
   - **Профиль** — чтобы вести рост и вес;
   - **Уведомления** — чтобы ничего не забывать.

## Если в Android-сборке белый экран

Самая частая причина: после изменений в React-коде были собраны только web-файлы, но они не были скопированы в Android-проект Capacitor.

Перед сборкой APK/AAB обязательно выполните:

```bash
npm run build:android
```

Эта команда:
- собирает свежий `dist`;
- синхронизирует его в `android` через `cap sync android`;
- обновляет плагины и web-assets для нативной сборки.
